# RBAC Route Permission Matrix (Frozen Contract)

Document status: Frozen
Contract version: 1.0.0
Effective date: 2026-04-13
Scope: Backend route mounts in server.js

## Authorization Rules

- Permission code format: resource:action (example: booking:update)
- Admin bypass: role=superadmin or permission *:* is allowed across permission checks
- Ownership rule: user-scoped endpoints require ownership unless bypass permission is explicitly granted
- Token source of truth: role and permissions are trusted only from validated JWT payload generated by backend

## Canonical Auth Contract

### POST /api/auth/login

Success:

- success
- message
- data.token
- data.user
  - id
  - unified_user_id (operator)
  - user_type
  - legacy_user_id (tourist/association compatibility)
  - username
  - email
  - role
  - permissions

JWT payload fields:

- sub
- user_type
- id
- unified_user_id (operator)
- legacy_user_id (tourist/association compatibility)
- username
- role
- permissions
- iat
- exp

Error contract:

- 401: missing/expired/invalid token
- 403: insufficient permission (includes required permission context)

**GET /api/users query parameters:**

| Parameter       | Example              | Description                                         |
| --------------- | -------------------- | --------------------------------------------------- |
| `search`        | `?search=John`       | Shortcut — searches `name` + `email` (LIKE)         |
| `q[field_pred]` | `?q[name_cont]=John` | Ransack predicate (cont, eq, gt, lt, in, etc.)      |
| `sort`          | `?sort=name_desc`    | Sort field + direction (or `?sort=name&order=DESC`) |
| `page`          | `?page=2`            | Page number (default: 1)                            |
| `per_page`      | `?per_page=5`        | Items per page (default: 10, alias: `limit`)        |
